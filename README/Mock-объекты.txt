Что такое mock-объекты?

Mock-объекты — это "поддельные" объекты, которые мы создаём для тестирования. Они позволяют изолировать тестируемый класс
от его зависимостей. Это значит, что мы можем проверить поведение класса без фактического выполнения кода его зависимостей.


Как создаются mock-объекты?

В Java с использованием библиотеки Mockito, mock-объекты создаются с помощью аннотации @Mock. Например:

        @Mock
        private HelloService helloService;

Здесь helloService — это mock-объект, который будет использоваться в тестах.


Как mock-объекты взаимодействуют с тестируемыми классами?

Тестируемый класс, например GreeterService, может зависеть от других классов, таких как HelloService. Чтобы протестировать
GreeterService, мы используем mock-объект HelloService, чтобы контролировать его поведение и проверить, как GreeterService реагирует на различные сценарии.


Пример взаимодействия:

    1. Создание mock-объекта:

        @Mock
        private HelloService helloService;

	2. Внедрение mock-объекта в тестируемый класс:

        @InjectMocks
        private GreeterService greeterService;

Здесь greeterService будет использовать mock-объект helloService.

	3. Настройка поведения mock-объекта:

В методе, помеченном аннотацией @BeforeEach, мы можем настроить, что mock-объект должен возвращать при вызове его методов:

        @BeforeEach
        void configureMock() {
           Mockito.doReturn("Hello").when(helloService).randomHello();
        }

Это значит, что каждый раз, когда randomHello() вызывается у helloService, он будет возвращать строку "Hello".

		4. Тестирование поведения:
Теперь мы можем писать тесты, чтобы проверить, как GreeterService ведёт себя с этим mock-объектом:

        @Test
        void whenNameIsGiven_ThenGreeterReturnsHelloName() {
        String name = "Test";
            Assertions.assertEquals("Hello, Test", greeterService.greet(name));
        }

Таким образом, mock-объекты позволяют нам изолировать тестируемый класс и проверить его поведение в различных ситуациях,
не полагаясь на фактическую реализацию его зависимостей.
